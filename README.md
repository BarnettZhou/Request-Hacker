# Request Hacker

一个轻量的 Chrome 请求拦截插件，用于在**前端层面**拦截 `fetch` / `XMLHttpRequest` 并替换返回数据，无需修改后端接口。

## 功能特性

- **双协议拦截**：同时拦截 `fetch` 和 `XMLHttpRequest` 请求
- **多响应配置**：每条规则支持多个响应配置，点击 tab 秒切不同返回值
- **通配符匹配**：支持 `*` 通配符 URL 匹配，灵活覆盖接口
- **规则管理**：增量导入/导出规则，支持 JSON 格式备份
- **实时生效**：保存规则后刷新页面即可生效，无需重启浏览器

## 适用场景

- 前后端联调时 mock 后端响应
- 模拟不同用户数据、异常状态码
- 本地开发时绕过某些权限校验
- 测试前端对不同响应数据的处理逻辑

## 安装方式

### 开发者模式加载（推荐）

1. 下载本项目源码
2. 打开 Chrome 扩展管理页面：`chrome://extensions/`
3. 右上角开启「开发者模式」
4. 点击「加载已解压的扩展程序」
5. 选择本项目的根目录文件夹

## 使用说明

### 1. 创建规则

- 点击浏览器工具栏上的插件图标打开配置页面
- 点击「新建规则」创建拦截规则
- 填写规则名称和 URL 匹配模式（支持 `*` 通配符）

### 2. 配置响应

- 在「响应配置」区域添加多个响应配置
- 每个配置可独立命名和填写 JSON 响应数据
- 点击不同配置 tab 可快速切换当前使用的返回值

### 3. 规则开关

- 点击规则卡片右上角的开关可启用/禁用该规则
- 在 popup 弹窗中也可直接切换规则开关和响应配置

### 4. 导入/导出

- 配置页面左下角支持导出规则为 JSON 文件
- 导入规则时同名规则会覆盖，不同名规则会追加

## 技术原理

Content Script 在页面加载时注入到 `MAIN` world，重写 `window.fetch` 和 `window.XMLHttpRequest`。当请求 URL 匹配规则时，直接返回配置的 JSON 数据，不再发送到真实服务器。

> ⚠️ 仅拦截前端发出的请求，真实网络请求未实际发送。开发调试专用，请勿用于生产环境作弊。

## 数据存储

所有规则数据保存在 Chrome 本地存储 (`chrome.storage.local`) 中，仅保存在本地，不会上传到任何服务器。
